第一章 基本概念

1.1 前言

web就是网页，分为两种：

  • 静态web：
    • 例如 html，css
    • 提供给所有人看的数据始终不会发生变化！
  • 动态web：
    • 例如 淘宝，以及大部分的其它网站
    • 提供给所有人看的数据会发生变化，在不同时间不同地点看到的信息各不相同！
    • 技术栈：Servlet/JSP，ASP，PHP

JavaWeb：在Java中，动态web资源开发的技术。

1.2 web应用程序

程序 = 算法 + 数据结构

web应用程序即为，可以提供浏览器访问的程序。

  • 所有能访问到的资源和页面，都是通过 URL（统一资源定位符） 定位到某个计算机上从而获取的。
  • 统一的 web 资源会被放在同一个 文件夹下，web应用程序 –> Tomcat：服务器。即编写完的web应用程序，若想被外界访问，需要由一个服务器来统一管理。

一个web应用有多个部分组成：

  • html, css, js
  • jsp, servlet
  • Java 程序
  • jar 包
  • 配置文件（Properties）

1.3 静态web

客户端向服务器端口80/443 发出请求某个 htm 或 html 文件的请求，服务器的 web服务 会监听该窗口，如果监听到了请求，会获取到该资源后响应给客户端。

缺点：

  • web页面无法动态更新，所有用户看到的都是同一个界面
  • 无法和数据库交互（数据无法持久化，用户无法交互）

1.4 动态web

页面可以动态展示。

缺点：

  • 加入服务器的动态web资源出现了错误，需要停机维护，重新编写后台程序，并重新发布。

优点：

  • web页面可以动态更新，不同用户看到不同的页面
  • 可以与数据库交互（数据持久化：商品信息，用户信息等可以持久获取）

第二章 web服务器

2.1 一些技术讲解

2.1.1 ASP

ASP：动态服务器页面（Active Server Page）

微软公司开发的代替CGI脚本程序的一种应用，它可以与数据库和其它程序进行交互国内最早流行。

在HTML中嵌入了VB的脚本，ASP开发中，基本每一个页面都有几千行的业务代码，页面很混乱，导致维护成本高。

ASP.NET 是 ASP 的升级，ASP.NET 的代码后置,即后台程序代码与前台页面代码相分离,而不像ASP那样前台后台的代码全放在一起。ASP.NET 是C#开发的web应用的运行环境。

2.1.2 PHP

号称自己是世界上最好的语言。

优点：

  • 开发速度快
  • 功能强大
  • 跨平台
  • 代码简单

缺点：

  • 无法承载大访问量

2.1.3 JSP/Servlet

JSP（全称Java Server Pages）是由[Sun Microsystems](https://baike.baidu.com/item/Sun Microsystems)公司主导创建的一种动态网页技术标准。JSP部署于网络服务器上，可以响应客户端发送的请求，并根据请求内容动态地生成HTML、XML或其他格式文档的Web网页，然后返回给请求者。JSP将Java代码和特定变动内容嵌入到静态的页面中，实现以静态页面为模板，动态生成其中的部分内容。

  • 是 sun 公司主推的 B/S 架构
  • 基于Java语言（所有大公司，或者一些开源的组件，都是用 Java 写的）
  • 可以承载高并发，高可用和高性能问题带来的影响
  • 语法非常像 ASP

2.2 web服务器

web服务器：用来处理用户的请求。

2.2.1 IIS

微软 web 服务器，内置在Windows系统中

2.2.2 Tomcat

Tomcat是Apache 软件基金会（Apache Software Foundation）的Jakarta 项目中的一个核心项目。最新的Servlet 和JSP 规范总是能在Tomcat 中得到体现。因为Tomcat 技术先进、性能稳定，而且免费，成为目前比较流行的Web 应用服务器。

Tomcat 服务器是一个免费的开放源代码的Web 应用服务器，属于轻量级应用服务器，在中小型系统和并发访问用户不是很多的场合下被普遍使用，是开发和调试JSP 程序的首选。

omcat 实际上运行JSP 页面和Servlet。目前Tomcat最新版本为10.0.5。

第四章 HTTP详解

4.1 什么是HTTP

HTTP（超文本传输协议，超文本就是指页面内可以包含图片、链接，甚至音乐、程序等非文字元素）是一个简单的请求响应协议，运行在TCP协议上。

4.2 两个版本

  • HTTP/1.0：非持续性连接
  • HTTP/1.1：持续性连接

4.3 HTTP请求

以访问 www.baidu.com 为例

Request URL: https://www.baidu.com/
Request Method: GET
Status Code: 200 OK
Remote Address: 36.152.44.95:443
Referrer Policy: strict-origin-when-cross-origin

Accept: text/html,application/xhtml+xml,application/xml;q=0.9,image/avif,image/webp,image/apng,*/*;q=0.8,application/signed-exchange;v=b3;q=0.9
Accept-Encoding: gzip, deflate, br
Accept-Language: zh-CN,zh;q=0.9
Connection: keep-alive
...省略
  • 请求行：请求方法（GET/POST/HEAD） + URL + 协议版本
    • get：请求能携带的参数比较少，且不安全，但高效
    • post：请求携带参数没有限制，不会在浏览器的URL地址显示数据内容，安全但不高效
  • 请求头：头部字段名： + 值
    • Accept：告诉浏览器，它支持的数据类型
    • Accept-Encoding：支持的编码格式 GBK UTF-8
    • Accept-Language：告诉浏览器它的语言环境
    • Cache-Control：缓存控制
    • Connection：告诉浏览器请求完是断开还是保持连接
    • HOST：主机….

4.4 HTTP响应

package com.kuang.servlet;

import javax.servlet.ServletException;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import java.io.IOException;
import java.io.PrintWriter;

public class HelloServlet extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        //这样在输出中文时不会乱码
        resp.setContentType("text/html;charset=utf-8");
        //或者可以直接用 resp.setCharacterEncoding("utf-8")
        PrintWriter writer = resp.getWriter();
        writer.print("Hello, Servlet");
    }

    @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        super.doPost(req, resp);
    }
}

  • 编写Servlet的映射

    为什么需要映射：因为需要通过浏览器访问便携的程序，而浏览器需要连接web服务器，所以要在web服务器中注册我们写好的Servlet程序，还需要给他一个浏览器可以访问的路径。

    在web.xml 中

<?xml version="1.0" encoding="UTF-8"?>
<web-app xmlns="http://xmlns.jcp.org/xml/ns/javaee"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee
                         http://xmlns.jcp.org/xml/ns/javaee/web-app_4_0.xsd"
         version="4.0"
         metadata-complete="true">
    <!-- register Servlet -->
    <servlet>
        <servlet-name>hello</servlet-name>
        <servlet-class>com.kuang.servlet.HelloServlet</servlet-class>
    </servlet>
    <!-- the path for request servlet-->
    <servlet-mapping>
        <servlet-name>hello</servlet-name>
        <url-pattern>/hello</url-pattern>
    </servlet-mapping>

</web-app>
  • 配置 Tomcat，注意要配置项目发布的路径

  • 启动测试，进入网址http://localhost:8080/s1/hello，可以看到 “Hello, Servelt”字样。

6.3 Servlet原理

Web服务器在收到浏览器请求之后，会调用Servlet的Service方法。

6.4 Mapping问题

  • 一个 Servlet 可以指定一个映射路径
    <servlet-mapping>
        <servlet-name>hello</servlet-name>
        <url-pattern>/hello</url-pattern>
    </servlet-mapping>
  • 一个 Servlet 可以指定多个映射路径
    <servlet-mapping>
        <servlet-name>hello</servlet-name>
        <url-pattern>/hello1</url-pattern>
    </servlet-mapping>
    <servlet-mapping>
        <servlet-name>hello</servlet-name>
        <url-pattern>/hello2</url-pattern>
    </servlet-mapping>
    <servlet-mapping>
        <servlet-name>hello</servlet-name>
        <url-pattern>/hello3</url-pattern>
    </servlet-mapping>
  • 一个 Servlet 可以指定通用映射路径
<servlet-mapping>
    <servlet-name>hello</servlet-name>
    <!-- hello/下的任何路径都对应到该Servlet-->
    <url-pattern>/hello/*</url-pattern>
</servlet-mapping>
  • 当前 Servlet 指定为默认请求路径
<servlet-mapping>
    <servlet-name>hello</servlet-name>
    <url-pattern>/*</url-pattern>
</servlet-mapping>
  • 自定义后缀实现请求映射
<servlet-mapping>
    <servlet-name>hello</servlet-name>
    <!-- 以zyt为结尾的路径都可以访问到该页面内容 -->
    <!-- 注意这里*前面不能写/ -->
    <url-pattern>*.zyt</url-pattern>
</servlet-mapping>

  • 优先级问题

    指定了固有的映射路径优先级最高，如果找不到会走默认请求。如果下面请求 /hello 路径，会显示 HelloServlet 类的内容，如果请求其他路径，会显示 ErrorServlet

   <servlet>
        <servlet-name>hello</servlet-name>
        <servlet-class>com.kuang.servlet.HelloServlet</servlet-class>
    </servlet>
    <!-- the path for request servlet-->
    <servlet-mapping>
        <servlet-name>hello</servlet-name>
        <url-pattern>/hello</url-pattern>
    </servlet-mapping>

    <!-- register Servlet -->
    <servlet>
        <servlet-name>error</servlet-name>
        <servlet-class>com.kuang.servlet.ErrorServlet</servlet-class>
    </servlet>
    <!-- the path for request servlet-->
    <servlet-mapping>
        <servlet-name>error</servlet-name>
        <url-pattern>/*</url-pattern>
    </servlet-mapping>

6.5 ServletContext

web容器在启动的时候，会为每个web程序创建一个对应的 ServletContext 对象（有且只有一个，对应单例模式），代表当前的 web应用。

也就是全局的，所有用户都可以共享的一些内容（PS：用户私有的内容存储在服务器的Session中）

  • 共享数据

    在这个Servlet类中用到的数据可以被同一个web程序下其他的Servlet读取到

  • 获取初始化的参数

    可以通过 web.xml 配置应用初始化参数：

  <context-param>
    <param-name>url</param-name>
    <param-value>jdbc:mysql://localhost:3306/mybatis</param-value>
  </context-param>

public class GetInit extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        ServletContext servletContext = this.getServletContext();
        String url = servletContext.getInitParameter("url");
        resp.getWriter().print(url);
    }
}

  • 请求转发

    通过下面的代码可以得到 /getu 界面，但是路径不会变，只会对页面请求一次（而重定向会改变路径，实质是两次请求）

public class GetDispatch extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        System.out.println("Entering GetDispatcher");
        ServletContext servletContext = this.getServletContext();
        servletContext.getRequestDispatcher("/getu").forward(req, resp);
    }
}
  • 读取资源文件
    • 在 java 目录下和 resources 目录下新建 properties 文件，发现最终都被打包到了 WEB-INF->classes下，这个路径一般称为 classpath。
    • 在properties里写如下内容，并通过 ServletContext的 getResourceAsStream获取到资源流，再用Properties对象进行读取
username=root
password=1234

package com.kuang.servlet;

import javax.servlet.ServletException;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import java.io.IOException;
import java.io.InputStream;
import java.util.Properties;

public class GetProperties extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        InputStream resourceAsStream = this.getServletContext().getResourceAsStream("/WEB-INF/classes/db.properties");
        Properties properties = new Properties();
       //加载流
        properties.load(resourceAsStream);
        String username = (String) properties.get("username");
        String password = (String) properties.get("password");
        resp.getWriter().print(username + ":" + password);
    }
}

7.1 会话

会话，指用户登录网站后的一系列操作，包括添加购物车、收藏夹、支付等其他动作。

7.2 两种保存会话的技术

为了保持用户登录的状态，有两个技术：Cookie与Session。Cookie是给客户端一个专属于个人的信息验证码，Session在服务器端记录用户的信息

  1. 存储位置不同: Cookie是保存在客户端（浏览器）， Session是保存到服务器端。
  2. 存储数据量大小不同: Cookie存储是有限的, 不超过4KB。而Seesion是无限制的（因为Session需要存储会话内容，这对容量要求比较高）

import javax.servlet.http.HttpServletResponse;
import java.io.IOException;
import java.io.PrintWriter;
import java.util.Date;

public class CookieDemo01 extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        //保存用户上一次访问的时间，可以通过服务器给客户端打包一个时间到对象里
        resp.setHeader("Content-type", "text/html;charset=utf-8");
        resp.setCharacterEncoding("utf-8");
        req.setCharacterEncoding("utf-8");
        PrintWriter writer = resp.getWriter();
        //Cookie是客户携带的验证码，服务器要从客户端获取的
        //可能有多个Cookie
        Cookie[] cookies = req.getCookies();
        //判断Cookie是否存在
        if(cookies != null) {
            //如果存在就要遍历数组
            writer.write("您上一次访问的时间是：");

            for(Cookie co : cookies) {
                if("lastLoginTime".equals(co.getName())) {
                    long lastLoginTime = Long.parseLong(co.getValue());
                    Date date = new Date(lastLoginTime);
                    writer.write(date.toLocaleString());
                }
            }

        } else {
            writer.write("这是您第一次访问本网站");

        }
        //服务器要给客户端分配Cookie
        Cookie cookie = new Cookie("lastLoginTime", System.currentTimeMillis() + "");
        resp.addCookie(cookie);
    }

    @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        doGet(req, resp);
    }
}

7.2.2 Session

服务器会给每个用户（浏览器）创建一个Session对象，只要浏览器没关闭且Session没过期，这个Session就会一直存在。

session其实分为客户端Session和服务器端Session。

  • 当用户首次与Web服务器建立连接的时候，服务器会给用户分发一个 SessionID作为标识。SessionID是一个由24个字符组成的随机字符串。用户每次提交页面，浏览器都会把这个SessionID包含在 HTTP头中提交给Web服务器，这样Web服务器就能区分当前请求页面的是哪一个客户端。这个SessionID就是保存在客户端的，属于客户端Session。

  • 其实客户端Session默认是以cookie的形式来存储的，所以当用户禁用了cookie的话，服务器端就得不到SessionID。这时我们可以使用url的方式来存储客户端Session。也就是将SessionID直接写在了url中，当然这种方法不常用。

session的创建

当程序需要为某个客户端的请求创建一个session时，服务器首先检查这个客户端的请求里是否已包含了sessionId，如果已包含则说明以前已经为此客户端创建过session，服务器就按照sessionId把这个session从后台数据库中检索出来使用（检索不到，会新建一个）。

如果客户端请求不包含sessionId，则为此客户端创建一个session并且生成一个与此session相关联的sessionId，并发送给浏览器，浏览器下次来访问的时候，会在request的cookie中带上sessionid，然后通过sessionid在服务器的缓存中获取实际数据。

Java Server Pages：Java服务器端页面，和Servlet一样，都是用于动态页面。

JSP 和 HTML 语法十分相像，他们的区别有：

第九章 JavaBean

一个实体公共类：

  • 必须提供一个无参构造函数。
  • 需要被序列化并且实现了 Serializable 接口。
  • 属性必须私有化
  • 要有一系列的 getter 或 setter 方法

一般用来和数据库的字段做映射 ORM（对象关系映射）：

  • 表—>类
  • 字段—>属性
  • 行记录—>对象

id name age

第十章 MVC三层架构

什么是MVC：

  • Model（模型）
  • View（视图） ：jsp
  • Controller （控制器）：也就是写的Servlet

其实Servlet和jsp都可以写java代码，但是Servlet专注于请求处理以及控制试图跳转，而jsp专注于显示数据。

10.1 早期架构

用户直接访问控制层，控制层可以直接操作（CRUD，增删改查）数据库。

servlet的代码中：处理请求、响应、视图跳转、处理JDBC、处理业务代码、处理逻辑代码。

弊端：JDBC的代码存储于Servlet中，使得程序臃肿，且不利于维护

如何解决？在架构中，没有什么是加一层解决不了的！

10.2 MVC三层架构

10.2.1 Model

  • 业务处理：业务逻辑（Service层）
  • 数据持久层：CRUD（Dao层）

10.2.2 View

  • 展示数据
  • 提供链接发起Servlet请求

10.2.3 Controller（Servlet）

  • 接收用户的请求：request请求参数、Session信息….
  • 交给业务层处理对应的代码
  • 控制视图跳转

以登录为例：

登录（视图按钮）—>接收用户的登录请求（视图发起请求给Servlet）—>Servlet 处理用户的请求（获取用户登录的参数，username，password）—->交给业务层 Service 处理登录业务（判断用户名密码是否正确：事务）—>Dao层查询用户名和密码是否正确–>从数据库调取查询数据

package com.kuang.filter;

import javax.servlet.*;
import java.io.IOException;

public class CharacterEncodingFilter implements Filter {

    //初始化：web服务器启动，就以及初始化了，因为它要一直监听，随时等待过滤对象出现！
    public void init(FilterConfig filterConfig) throws ServletException {
        System.out.println("CharacterEncodingFilter初始化");
    }

    //Chain : 链
          /*
          1. 过滤中的所有代码，在过滤特定请求的时候都会执行
          2. 必须要让过滤器继续同行
              chain.doFilter(request,response);
           */
    public void doFilter(ServletRequest servletRequest, ServletResponse servletResponse, FilterChain filterChain) throws IOException, ServletException {


        servletRequest.setCharacterEncoding("utf-8");
        servletResponse.setCharacterEncoding("utf-8");
        servletResponse.setContentType("text/html;charset=UTF-8");

        System.out.println("CharacterEncodingFilter执行前");
        //让我们的请求继续走，如果不写，程序到这里就被拦截停止！
        filterChain.doFilter(servletRequest, servletResponse);
        System.out.println("CharacterEncodingFilter执行后");
    }

    //销毁：web服务器关闭的时候，过滤器会销毁
    public void destroy() {
        System.out.println("CharacterEncodingFilter销毁");
    }
}

第十二章 Listener 监听器

本质就是要实现监听器的接口

下面以统计网站在线人数为例：

  • 实现监听器接口，并编写代码
package com.kuang.listener;

import javax.servlet.ServletContext;
import javax.servlet.http.HttpSessionEvent;
import javax.servlet.http.HttpSessionListener;

//统计网站在线人数 ：统计session
public class OnlineCountListener implements HttpSessionListener {
    //创建session监听： 看你的一举一动
    //一旦创建Session就会触发一次这个事件！
    public void sessionCreated(HttpSessionEvent httpSessionEvent) {
        ServletContext servletContext = httpSessionEvent.getSession().getServletContext();
        Integer onlineCount = (Integer) servletContext.getAttribute("OnlineCount");

        if(onlineCount == null) {
            onlineCount = 1;
        } else {
            onlineCount = onlineCount + 1;
        }

        servletContext.setAttribute("OnlineCount", onlineCount);
    }

    //销毁session监听
    //一旦销毁Session就会触发一次这个事件！
    /*
    Session销毁：
    1. 手动销毁  getSession().invalidate();
    2. 自动销毁 去web.xml配置<session-config>下的<session-timeout>,以分钟为单位
     */
    public void sessionDestroyed(HttpSessionEvent httpSessionEvent) {
        ServletContext servletContext = httpSessionEvent.getSession().getServletContext();
        Integer onlineCount = (Integer) servletContext.getAttribute("OnlineCount");

        if(onlineCount != null) {
            onlineCount = onlineCount - 1;
        }

        servletContext.setAttribute("OnlineCount", onlineCount);

    }
}
  • 在web.xml注册监听器
<!--注册监听器-->
<listener>
    <listener-class>com.kuang.listener.OnlineCountListener</listener-class>
</listener>
  • 编写jsp界面
<body>

  <h1>
    当前有 <span><%=this.getServletConfig().getServletContext().getAttribute("OnlineCount")%></span> 人在线
  </h1>
    
</body>
