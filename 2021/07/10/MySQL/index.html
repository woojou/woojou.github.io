数据库概述

一些基本概念区分

DB：DataBase（数据库，数据库实际上在硬盘上以文件形式存在）

DBMS：DataBase Management System（数据库管理系统，常见的有MySQL、Oracle、DB2、Sybass、SqlServer）

SQL：结构化查询语言，是一门标准通用语言，标准的SQL适合于所有的数据库产品。SQL属于高级语言。SQL语言在执行时，内部会先进行编译(由DBMS完成)再执行。

综上，DBMS负责执行sql语句，通过执行sql语句来操作DB当中的数据。

安装

安装在 C:\Program Files (x86)\MySQL

计算机右键点管理，点服务，里面有MySQL，右键属性，里面是自动的，证明这个服务是自启的，可以改成手动。服务端口是3306。

  2. 输入命令：mysql -uroot -p
  3. 回车后输入密码，即可完成登录操作
  4. 输入exit退出

表

数据库的基本组成单元，所有的数据都以表格形式组织，可读性强。

包括行和列：行是数据/记录(data)；列是字段(column)

字段包括那些属性？

  • 字段名：例如叫学号、姓名、年龄
  • 数据类型：例如varchar、int
  • 相关约束：例如字段不能为空等约束
学号(int) 姓名(varchar) 年龄(int)
110 张三 20
120 李四 21

SQL语句分类

  • DQL（数据查询语言 Data Query Language）：查询语句，凡是select的语句都是DQL
  • DML（数据操作语言 Data Manipulation Language）：insert、 delete、 update，对表中数据进行增删改
  • DDL（数据定义语言 Data Definition Language）：create、 drop、 alter，对表中结构增删改
  • TCL（事务控制语言 Transactional Control Language）：commit提交事务，rollback回滚事务
  • DCL（数据控制语言 Data Control Language）：grant授权、revoke撤销权限等

分组

分组函数一共五个，也称为多行处理函数，即输入多行输出一行。相应的，也会有单行处理函数，所谓单行处理函数就是输入一行输出一行。

​ 举例单行处理：计算每个员工的年薪？

​ select ename, (sal+comm) * 12 as yearsal from emp;

mysql> select ename, (sal+comm) * 12 as yearsal from emp;+--------+----------+| ename  | yearsal  |+--------+----------+| SMITH  |     NULL || ALLEN  | 22800.00 || WARD   | 21000.00 || JONES  |     NULL || MARTIN | 31800.00 || BLAKE  |     NULL || CLARK  |     NULL || SCOTT  |     NULL || KING   |     NULL || TURNER | 18000.00 || ADAMS  |     NULL || JAMES  |     NULL || FORD   |     NULL || MILLER |     NULL |+--------+----------+

观察上面的单行处理函数，发现只要在计算时遇到了NULL，那么最终结果就是NULL。即单行处理函数无法忽略NULL。为了避免这样的情况可以用空处理函数：ifnull()，仍然以上面的为例，就可以写成select ename, (sal+ifnull(comm,0)) * 12 as yearsal from emp;

连接查询

什么是连接查询

在实际开发中，大部分情况下都不是从单表中查询数据，一般都是多张表联合查询取出最终结果。

连接查询的分类

按年代划分：

  • SQL92 （一些老的DBA可能还在使用。Database Administrator：数据库管理员）
  • SQL99（比较新的语法）

根据表的连接方式：

  • 内连接

外连接

和内连接的区别？

  • 内连接：假设A表和B表连接，使用内连接的话，凡是A表和B表能够匹配上的记录查询出来，这就是内连接。AB两张表没有主副之分，两张表是平等的。
  • 外连接：假设A表和B表连接，使用外连接的话，AB两张表中有一张是主表，一张是副表，主要查询主表中的数据，捎带着查询副表。当副表中的数据没有和主表中的数据匹配上，副表自动模拟出NULL与之匹配。

外连接的分类：

  • 左外连接（左连接）：表示左边的表是主表

  • 右外连接（右连接）：表示右边的表是主表

    左连接有右连接的写法，右连接也会有对应的左连接的写法。

​ select b.ename ‘领导’, a.ename ‘员工’ from emp b right join emp a on a.mgr = b.empno;

char 和 varchar怎么选择？

​ 如果规定是char(size)，那么就算存入的字符串本身长度小于size，也会分配给它size的空间。如果规定是varchar(size)，会根据存入字符串的长度动态分配。但是用上面两种数据类型存储的字符串长度都不能超过size。

约束 Constraint（重点）

什么是约束？常见的有哪些？

在创建表的时候，给表的字段添加相应的约束，使保证表中数据的合法性、有效性、完整性。

常见的约束有：

  • 非空约束（not null）：约束字段不能为NULL
  • 唯一约束（unique）：约束字段不能重复
  • 主键约束（primary key）：约束字段既不能为NULL，也不能重复，简称PK
  • 外键约束（foreign key）：简称FK
  • 检查约束（check）：Oracle数据库有check约束，但是mysql没有

非空约束 not null

create table t_user(    id int,    username varchar(255) not null,    password varchar(255));

如果执行” insert into t_user(id, password) values (1,’123’); “ 会报错：ERROR 1364 (HY000): Field ‘username’ doesn’t have a default value

唯一约束 unique

唯一约束修饰的字段具有唯一性，不能重复，但可以为NULL。

drop table t_user;create table t_user(    id int,    username varchar(255) unique //【列级约束】);insert into t_user (id, username) values (1, 'zhangsan');insert into t_user (id, username) values (2, 'zhangsan');

报错：ERROR 1062 (23000): Duplicate entry ‘zhangsan’ for key ‘username’

案例：给两个列或多列添加unique

drop table t_user;create table t_user(    id int,    usercode varchar(255),    username varchar(255),    unique(usercode, username) //usercode和username联合唯一【表级约束】);

insert into t_user (id, usercode, username) values (1, ‘111’, ‘zhangsan’);
insert into t_user (id, usercode, username) values (2, ‘222’, ‘zhangsan’);

以上不会报错。

如果分别加unique，上面的代码是会报错的。

  • 主键字段中的数据不能为NULL，也不能重复。

  • 主键相关术语：

    • 主键约束：primary key
    • 主键字段：带有主键约束的字段
    • 主键值：主键字段中插入的值

  • 主键的作用？

    表的设计三范式中有要求，第一范式就要求任何一张表都应该有主键。

    主键列的记录代表每一行中的记录在表中标识着唯一性，就像人的身份证号码一样。

  • 一张表的主键约束只能有一个！

  • 主键分类？

    根据主键字段和字段数量来划分：单一主键（推荐，常用） 和 复合主键（多个字段联合起来添加一个主键的约束，不推荐使用，违背了三范式）。

    根据主键性质来划分：自然主键（主键最好就是一个和业务没有任何关系的自然数） 和 业务主键（主键值和系统的业务挂钩，不推荐使用）。

  • 使用表级约束方式定义主键：

    drop table t_user;create table t_user(    id int,    username varchar(255),    email varchar(255),    primary key(id) //【表级约束】);

常见的存储引擎

MyISAM

不支持事务。

是MySQL最常用的存储引擎，但不是默认的。

它管理的表具有以下特征：

​ – 使用三个文件组织一张表：

​ • 格式文件 — 存储表结构的定义（mytable.frm）

​ • 数据文件 — 存储表中的数据（mytable.MYD）

​ • 索引文件 — 存储表中的索引（mytable.MYI）

​ – 灵活的AUTO_INCREMENT字段处理

​ – 可被转换为压缩、只读表来节省空间提高检索效率

InnoDB

支持事务、行级锁、外键，这种存储引擎的安全得到保障

– 表的结构存储在frm文件中

– 数据存储在tablespace这样的表空间中（逻辑概念），无法被压缩，无法转换成只读。

– 在MySQL服务器崩溃后提供自动恢复

– 支持外键及引用的完整性，包括级联删除和更新 （删父的时候，用过父这条数据的子也会被删除）

MEMORY存储引擎

不支持事务，数据容易丢失，因为所有数据和索引都是存储在内存当中，但也因此查询速度最快。

以前叫做HEAP引擎。

– 在数据库目录内，每个表均以.frm格式的文件表示。

– 表数据及索引被存储在内存中。

– 表级锁机制。

– 不能包含TEXT或BLOB字段。

事务（Transaction）

什么是事务？

  • 一个事务是一个完整的业务逻辑单元，不可再分

    比如：银行账户转账，从A账户向B账户转账10000，需要执行两条update语句，一条是从A账户中减去10000，一条是给B账户加上10000。而这两条DML要求必须同时成功或同时失败，这需要使用”事务机制”。

  • 和事务相关的语句只有DML，即只有insert、delete、update，这三个语句都是和数据库中的”数据“有关的，事务的存在是为了保证数据的完整性、安全性。

事务的特性ACID

  • A：Atomicity，原子性。事务是最小的工作单元，不可再分。

  • C：Consistency，一致性。事务必须保证多条DML同时成功或同时失败。

  • I：Isolation，隔离性。事务A与事务B之间相互隔离。

    事务隔离性存在隔离级别，理论上隔离级别包括四个：

    • 第一级别：读未提交（read uncommitted）

      对方事务还没有提交，我们当前事务可以读取到对方没有提交的数据。

      读未提交存在脏读（Dirty Read）现象，指读到脏数据。

    • 第二级别：读已提交（read committed）

      对方事务提交之后的数据我方可以读取到。

      这种隔离级别解决了脏读现象。

      读已提交存在的问题是：不可重复读。

    • 第三级别：可重复读（repeatable read）

      解决了不可重复读问题。

      存在的问题是：读取到的数据是幻象。

    • 第四级别：序列化读/串行化读

      解决了所有问题。

      但是效率低。

    Oracle默认的隔离级别是读已提交。

    MySQL默认的隔离级别是可重复读。

  • D：Durability，持久性。最终数据必须持久化到硬盘文件中，事务才算成功地结束。

索引

什么是索引

索引就相当于是一本书的目录，通过目录可以快速找到对应资源。

数据库里，在查询一张表时有两种检索方式：全表扫描和根据索引扫描。

索引提高检索效率的原因：缩小了扫描的范围。

不可以随意地添加索引，因为索引也是数据库中的对象，需要数据库不断维护，有维护成本。如果表中的数据需要经常被修改，这就不适合添加索引，因为数据一旦修改，索引需要重新排序，进行维护。

添加索引是给某个字段或某些字段添加，主键和具有unique约束的字段自动会添加索引。

索引实现原理

索引底层采用的数据结构是B + Tree，底层索引进行了排序、分区，索引会携带数据在表中的“物理地址”，最终通过索引检索到数据后，获取到关联的物理地址，通过物理地址定位到表中的数据。

索引的分类

单一索引：给单个字段添加索引

复合索引：给多个字段联合添加索引

主键索引：主键会自动添加索引

唯一索引：有unique约束的字段会自动添加索引

如何添加索引

创建：

​ create index 索引名 on 表名（字段名）;

删除：

​ drop index 索引名称 on 表名;

添加索引的场景

  • 数据量庞大（根据客户需求和线上环境）

  • 字段很少参与DML操作（字段进行修改时，索引也需要维护修改）

  • 字段经常出现在where子句中 （经常根据哪个字段查询）

索引失效

模糊查询的时候，第一个通配符用的是%，这个时候索引会失效。

视图

什么是视图

同一张表用不同角度去看待。

对视图进行增删改查，会影响到原数据。也可以对视图进行CRUD操作。

视图的作用：视图可以隐藏表的实现细节，保密级别比较高的系统，数据库只对外提供相关的视图。

创建与删除视图

创建：

​ create view 视图名 as select 字段名1，字段名2，… from 表名;

删除：

​ drop view 视图名;

注意：只有DQL语句才能以视图对象的方式创造出来。

数据库设计三范式（重点）

什么是设计范式

设计表的依据，按照三范式设计的表不会出现数据冗余。

三范式

  • 第一范式：任何一张表都应该有主键，并且每个字段原子性不可再分。

  • 第二范式：建立在第一范式的基础之上，所有非主键字段完全依赖主键，不能产生部分依赖。

    设计口诀：多对多，三张表，关系表中两个外键。

  • 第三范式：建立在第二范式的基础之上，所有非主键字段直接依赖主键，不能产生传递依赖。

    设计口诀：一对多，两张表，多的表加外键。

注意：在实际开发中，以满足客户需求为主，有时会拿冗余换执行速度。

一对一设计的方案：

  1. 主键共享
  2. 外键唯一

​ select deptno, max(sal) as maxsal from emp group by deptno;

5. 取得平均薪水最高的部门的部门编号

第一种：

​ 第一步：找出每个部门的平均薪水 select deptno, avg(sal) from emp group by deptno

​ 第二步：可以直接降序select deptno, avg(sal) as avgsal from emp group by deptno order by avgsal desc limit 1

第二种：

​ 第一步：找出每个部门的平均薪水 select deptno, avg(sal) from emp group by deptno

​ 第二步：用Max找到最大值 select max(t.avgsal) from (select deptno, avg(sal) as avgsal from emp group by deptno) t

​ 第三步：找到平均薪水等于max的最大值的部门号select deptno, avg(sal) as avgsal from emp group by deptno having avgsal = (select max(t.avgsal) from (select deptno, avg(sal) as avgsal from emp group by deptno) t)

6. 取得平均薪水最高的部门的部门名称

第一步：找出每个部门的平均薪水 select deptno, avg(sal) from emp group by deptno

第二步：可以直接降序取最高部门号select deptno, avg(sal) as avgsal from emp group by deptno order by avgsal desc limit 1

第三步：连接部门表select d.dname, avg(e.sal) as avgsal from emp e join dept d on e.deptno = d.deptno group by d.dname order by avgsal desc limit 1

7. 求平均薪水的等级最低的部门的部门名称

第一步：找出每个部门的平均薪水 select deptno, avg(sal) from emp group by deptno

第二步：通过表连接找出部门名称对应等级select s.grade, d.dname from salgrade s join (select deptno, avg(sal) as avgsal from emp group by deptno) t on t.avgsal between s.losal and s.hisal join dept d on t.deptno = d.deptno
